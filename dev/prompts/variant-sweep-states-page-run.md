## Operating rule for the sweep (a clarification this run installs)

The doctrine says variants are declared for states "the mockup does not currently show," but does not settle per-instance versus per-component. Operate **per-slug**: a state counts as *shown* if any rendered instance of that slug displays it in the mockup's default booted state (judge against the live DOM after `boot()`, not static markup alone — many instances are JS-built). Only states hidden at the slug level get declared. Placement: for static markup, declare on the first/most canonical instance of the slug (the same convention the label overlay uses to anchor its one-label-per-slug); for builder-emitted slugs, emit from the builder, so every generated instance carries the declaration uniformly — the duplication is harmless, the Variant-check dedupes. If one slug genuinely spans sub-forms with different state sets (for example `identity-badge` and its subagent mini form), declare the union on the canonical root and flag the split in your report; do not invent new slugs to resolve it. If dynamic workflows fan the sweep across subagents, each one judges against this same per-slug rule and the DESIGN.md ladder, and a single coordinating pass reviews the merged declarations for uniformity before the stage-1 commit — consistent calls matter more than parallel speed. In stage 3, record this per-slug rule as one clarifying sentence in DESIGN.md's Variant declarations section (operator-approved at prompt review).
