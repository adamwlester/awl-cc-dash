## A. GLOBAL AESTHETIC DIRECTION  (preference — you decide feasibility)
The intent is to move the dashboard toward the cleaner, more **rounded card feel** of
`prompt-panel-action-strip.html` and `palette-options/index.html`, while keeping the major
three-column structure coherent.
- **Preference:** more rounding overall; adopt that card-style framing (rounded outer
  corners, a clean header bar, body, and footer strip) for panels; apply that treatment
  notably to **header content (heading strips) and the footer.**
- **Constraint:** it must **cohere as a whole** — don't round one element in isolation. If
  converting every element to this aesthetic breaks the layout or looks inconsistent, scale
  back and propose where radii should and shouldn't land. This direction is still being
  sorted, so **treat §A as a direction to prototype and report back on, not a locked spec.**
- **Icon-button convention (app-wide):** standardize icon buttons as **ghost buttons that are
  faintly visible at rest** (so available actions are discoverable) and emphasized on hover —
  like the existing inline "X". Apply consistently. A shared component/icon library is preferred
  especially if it improves coherence. Don't add heavy custom machinery just to restyle minor 
  controls.
- **Placement of repeated elements is yours to optimize.** The exact position/layout of recurring
  controls — clear/copy icon buttons, the various toggles and toggle groups, even the agent toggle
  strips (§F) — is a **default, not a mandate**. Where your judgment (UX, clean design, platform
  conventions, everything else in this brief) says a different placement reads better or is more
  standard, use it — then keep each control type **consistent with itself** across the app.
