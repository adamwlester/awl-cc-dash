# Auth Module Remediation Plan
## Critical Findings
1. **Token Expiry Bypass** (Severity: CRITICAL)
- `validateToken()` in refresh.ts accepts expired tokens
- No `exp` claim check before issuing new token
- Fix: Add `jwt.verify()` with `ignoreExpiration: false`
2. **Weak Secret Leak** (Severity: HIGH)
- Default dev secret `"changeme"` in config.defaults.ts
- No env var override enforced in production
- Fix: Require JWT_SECRET env var, fail startup if missing
3. **No Rate Limiting** (Severity: MEDIUM)
- Refresh endpoint has no throttling
- Fix: Add express-rate-limit to /auth/refresh
